# GitHub Issue 利用ガイド (テスター向け)

本演習では、発見したバグの報告に **GitHub Issue** を使用します。
開発者（講師・採点者）に正確にバグを伝えるため、以下のルールに従って起票してください。

## 1. Issueの作成手順
1. リポジトリの **[Issues]** タブをクリックします。
2. **[New issue]** ボタン（緑色）をクリックします。
3. 用意されている **"Bug Report"** テンプレートの [Get started] を選択します。

## 2. 良いバグ報告の書き方
テンプレートに沿って記述しますが、特に以下の点に注意してください。

### タイトル (Title)
一目で内容がわかるように書いてください。
* **悪い例**: バグがあります、動きません
* **良い例**: [書籍検索] シングルクォートを入力するとInternal Server Errorが発生する
* **良い例**: [貸出] 5冊借りている状態で6冊目が借りられてしまう

### 再現手順 (Steps to Reproduce)
誰がやっても同じ結果になるよう、箇条書きで具体的に書きます。
1. `user01` でログインする
2. 書籍一覧画面を開く
3. ...

### 期待する動作 vs 実際の動作
* **期待する動作**: 本来どうなるべきだったか（仕様書の記述など）。
* **実際の動作**: 実際に何が起きたか（エラー画面が出た、データが消えたなど）。

### 証拠 (Screenshots / Logs)
* エラー画面のスクリーンショットや、コンソールログのコピペがあれば必ず貼ってください。

## 3. ラベル (Labels) の活用
報告するバグの性質に合わせてラベルを付与してください（権限がある場合）。
* `bug`: 一般的なバグ
* `critical`: システムが停止する、データが消えるなどの重大なバグ
* `stipulated-bug`: (演習用) もし「仕込みバグ」だと確信した場合に使用（※講義ルールによる）

---

## 4. Issue テンプレート例
本プロジェクトには以下のテンプレートが設定されます。

```markdown
---
name: バグ報告 (Bug Report)
about: 発見した不具合を報告してください
title: "[機能名] バグの概要"
labels: bug
assignees: ''
---

## バグの概要
ここに不具合の要約を記載してください。

## 再現手順
1. '...' に移動する
2. '...' をクリックする
3. '...' と入力する
4. エラーを確認する

## 期待される動作
本来どう動くべきかを記載してください。

## 実際の動作
実際に起きた現象を記載してください。

## スクリーンショット
あればここに貼り付けてください。

## 環境
 - OS: [e.g. Windows 11]
 - Browser: [e.g. Chrome 120.0]
```
