# テスター体験に必要なドキュメント一覧

一貫実技演習において、テスター（演習参加者）が「仕事としてのテスト」を疑似体験するために必要なドキュメント類です。

## 1. 業務インプット資料（これが正解基準となる）
テスターがテストケースを作成したり、期待値を判断したりするために必須となる資料です。

- **機能要件定義書 / 基本仕様書**
    - 画面ごとの機能説明
    - 画面遷移図
    - 項目定義（入力制限、必須項目など）
    - 権限仕様（管理者と一般ユーザーの違い）

- **データベース設計書 (ER図・テーブル定義書)**
    - テーブル構造、リレーション
    - 「削除時の挙動」や「データの整合性」を裏取りするために必要

## 2. テスト業務遂行用の雛形
テスターが成果物を作成するためのフォーマットです。

- **テスト観点一覧表 (テンプレート)**
    - 何をテストするか（正常系、異常系、境界値など）を洗い出すためのリスト
    - ※今回は「サンプル」として一部記入済みのものを用意し、残りを埋めさせる形式も有効

- **テスト仕様書 / テストケース (テンプレート)**
    - テストNo、確認手順、期待値、実施日、結果、備考 などの欄があるExcel/Markdown表

- **GitHub Issues (バグ管理)**
    - テストで見つけたバグは、ExcelやRedmineではなくGitHubのIssue機能を使って報告させます。
    - **Issue Template**: 報告内容（再現手順、期待値など）を統一するための `.github/ISSUE_TEMPLATE` 設定。
    - **Issue利用ガイド**: 初心者向けに、Issueの書き方やラベルの使い方をまとめた資料。

## 3. テスト実施補助データ
- **テスト用マスターデータ定義**
    - テストに利用できる初期ユーザーID/Password
    - 初期登録されている書籍リスト（ISBNや在庫数）

---
### ※本プロジェクトでの作成優先度
今回はシステム開発側（我々）が、これらのドキュメントの「簡易版」を用意する必要があります。

1. **基本仕様書（README等に記載）** [最優先]
2. **テーブル定義（schema.sql 相当の情報）** [優先]
3. **テスト用アカウント情報** [必須]
4. **観点一覧・テストケース・バグ票** [演習フォーマットとして準備]
